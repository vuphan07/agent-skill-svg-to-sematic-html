# convert-svg-html

Convert a design‑tool **SVG export** (a single giant `<svg>`, usually wrapped in an
`.html` file, with base64 images, `<filter>` shadows and `<text>/<tspan>` text)
into clean, **semantic HTML/CSS** — then verify the rebuild against the original,
section by section.

Powered by the bundled Claude Code skill in
[`.claude/skills/svg-to-semantic-html/`](.claude/skills/svg-to-semantic-html/SKILL.md).

---

## What it does

- **Extract** embedded images & icons from the export.
- **Measure** exact design tokens, the layout spine, and line breaks.
- **Rebuild** as either:
  - **Flat** — pixel‑perfect, absolutely‑positioned draft (`transpile_flat.py`), or
  - **Semantic** — real `<header>/<section>/<p>/<form>/<footer>`, document flow
    (hand‑built from the analysis; this is the usual goal).
- **Verify** deterministically: render source + rebuild, diff them, and flag
  gradient/tint, missing rules, and size deltas — with an optional Gemini layer.

---

## Requirements

| Tool | Install | Used by |
|------|---------|---------|
| Python 3 + Pillow + numpy | `pip install pillow numpy` | analyze / stitch / visual_diff |
| Node + Puppeteer | `npm i` (or `npm i puppeteer`) | rendering to PNG |
| *(optional)* google‑genai + `GEMINI_API_KEY` | `pip install google-genai` | AI verify layer (`--ai`) |

- Set `CHROME_PATH` to reuse an installed Chrome/Chromium instead of Puppeteer's bundled one.
- On Windows, run Python with UTF‑8 to handle umlauts:
  `set PYTHONUTF8=1 && set PYTHONIOENCODING=utf-8` (or `export …` in bash).

## Project layout

```
svg-input/        # SOURCES  — the .html / .svg exports to convert
html-output/      # RESULTS  — one folder per source, SAME base name
                  #   html-output/<name>/index.html   (CSS + JS inline) + assets/
                  #   html-output/<name>/<name>.standalone.html  (assets base64 too — optional)
compare/          # generated by batch_check.py — comparison strips + diff_report.json
.claude/
  skills/svg-to-semantic-html/   # the converter skill (scripts + docs)
  settings.json                  # shared permissions (committed)
  settings.local.json            # personal overrides (git-ignored)
```

> **Convention (important):** sources and results are matched **by base name**.
> `svg-input/13.html` → `html-output/13/index.html`. Full rules in the skill's
> [`SKILL.md`](.claude/skills/svg-to-semantic-html/SKILL.md).

## Quick start

```bash
# 0. one-time
npm i
pip install pillow numpy            # + google-genai for the optional --ai layer

# 1. put your export in svg-input/, e.g. svg-input/15.html

# 2. extract assets + measure (writes html-output/15/)
python .claude/skills/svg-to-semantic-html/scripts/extract_assets.py svg-input/15.html html-output/15
python .claude/skills/svg-to-semantic-html/scripts/analyze_svg.py    svg-input/15.html --json html-output/15/analysis.json

# 3a. flat draft (instant, pixel-perfect reference)
python .claude/skills/svg-to-semantic-html/scripts/transpile_flat.py svg-input/15.html html-output/15-flat

# 3b. semantic build = html-output/15/index.html  (CSS + JS inline, one file; per SKILL.md)

# 4. verify every matched pair: render + stitch + deterministic diff
python .claude/skills/svg-to-semantic-html/scripts/batch_check.py
python .claude/skills/svg-to-semantic-html/scripts/batch_check.py --ai   # + Gemini labelling

# 5. (optional) one fully-portable file — base64-embed the images too
python .claude/skills/svg-to-semantic-html/scripts/inline_build.py html-output/15 --embed-assets
```

Browse a source in a browser:
`python .claude/skills/svg-to-semantic-html/scripts/serve.py svg-input 5501`

## Verifying a build

`batch_check.py` (or `visual_diff.py` for one pair) renders both pages and writes
to `compare/<name>/`:

- `compare_*.png` — **side‑by‑side strips** (source left, rebuild right).
- `diff_report.json` — machine‑readable findings: **page width/height + height delta**,
  per‑strip **tint delta** (catches over‑saturated gradients / wrong tint), and a
  **rule tally** (catches missing separators/accents).

**Trust the deterministic diff first.** Fix every `HIGH`, eyeball every `MED`.
`--ai` adds Gemini's semantic labels per flagged crop — treat them as *leads to
verify*, not facts. The diff is drift‑robust, so a 1–2px offset won't flag; use the
native‑resolution crops for those.

> **Font caveat:** exports target **Segoe UI**. Without it the fallback is wider,
> so text wraps differently and the page grows taller — a positive **height delta**
> is usually the missing font, not a layout bug. Keep `'Segoe UI'` first in the
> stack; embed a metric‑compatible font (e.g. Selawik) for cross‑platform fidelity.

## Optional AI layer (`--ai`)

Self‑contained — no external skill needed; `scripts/gemini_vision.py` is a small
vendored Gemini client.

1. `pip install google-genai`
2. Add your key (never commit it):
   ```bash
   cp .claude/skills/svg-to-semantic-html/.env.example .claude/skills/svg-to-semantic-html/.env
   # then edit .env → GEMINI_API_KEY=...
   ```
   Key is also read from a project‑root `.env` or `~/.claude/.env`.

The whole pipeline runs **offline** without a key; `--ai` only adds labelling on
already‑flagged crops.

## Sharing / git notes

- **Committed:** the skill (`.claude/skills/…`), `.claude/settings.json` (shared
  permission allow‑list), `svg-input/` (work inputs).
- **Git‑ignored:** `.env` (secrets), `.claude/settings.local.json` (personal),
  `node_modules/`, `compare/` and `html-output/` (regenerable — uncomment
  `html-output/` in `.gitignore` if you want to ship built pages).

  `.claude/settings.json`; keep that in your per‑user `~/.claude/settings.json`.
- Each teammate supplies their own `GEMINI_API_KEY`.
